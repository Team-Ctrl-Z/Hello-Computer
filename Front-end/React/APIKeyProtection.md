# 웹 클라이언트 보안 개선
1. 문제상황
칸그루맵 프로젝트에서 외부 API(HOT PEPPER API)를 사용하여 음식점 정보를 불러오는 기능을 구현했습니다.
처음에는 클라이언트에서 직접 API를 호출하려고 했지만, API 키가 공개되는 문제가 발생할 수 있다는 점을 인식했습니다.

2. 분석과정
React는 클라이언트에서 실행되기 때문에 .env에 저장된 키도 빌드 시점에 노출된다는 점을 확인했습니다.
실제 번들 파일을 분석해보면 누구나 API 키를 확인하고 악용할 수 있는 구조였습니다.

3. 원인
클라이언트에서 외부 API를 직접 호출하면, 키가 코드에 포함되어 보안상 취약한 구조가 됩니다.

4. 개선조치
API 키는 클라이언트에 절대 노출되지 않도록, 서버 측에서 외부 API를 대신 호출하는 프록시 구조로 변경했습니다.
React 앱에서는 /api/hotpepper 라는 자체 경유 API를 통해 간접적으로 호출하며,
실제 키는 서버 환경 변수에 저장하여 안전하게 보호했습니다.

5. 기대결과
API 키가 외부에 노출되지 않아 보안이 강화되었고

사용자의 API 오용이나 쿼터 초과 같은 운영 리스크를 사전에 방지할 수 있었습니다

보안성을 높이면서도 클라이언트는 단순한 경로로 데이터를 요청할 수 있어 유지보수성과 구조 안정성도 향상되었습니다
